## Critical Issues (Fix Immediately)

**1. Entity collision is the single biggest GEO risk.**
"Obrador Web" tokenizes near "obrador" (bakery, generic Spanish for bakeshop) and López Obrador (former Mexican president). AI models will preferentially resolve "Obrador" + "Madrid" + coffee context to **Obrar Madrid** bakery, not your agency. Disambiguation must be lexical, not just visual footer text.
*Fix:* Update `<title>` and llms.txt H1 to "Obrador Web — agencia diseño web cafeterías Madrid" (already partially done in footer); add explicit disambiguator in pricing post intro: "Obrador Web (no confundir con Obrar Madrid bakery)".

**2. Zero proof of delivery. All 8 portfolio entries flagged "concepto, no es trabajo oficial".**
Visitors learn the agency has never shipped a real client. Biggest conversion AND E-E-A-T blocker. This destroys Authority signals AI models look for.
*Fix:* Reframe as `/metodologia/` + structured "design briefs" (Reto / Decisiones / Métricas esperadas / Stack). Process transparency substitutes for missing track record. Once Pascal Café ships, replace 1 disclaimer with real case study.

## High Priority Issues (Fix Within 1 Week)

**3. Mobile LCP 5.4-5.7s on `/proyectos/`** fails Google's "Good" CWV threshold (<=2.5s). Hurts ranking on the page where prospects judge portfolio. Likely cause: 6+ portfolio hero images loading even with WebP+lazy.
*Fix:* Eager-load only first 2-3 above-fold; shrink mobile WebP variants to ~600px wide; preload first hero. ~1h.

**4. No Wikidata item for "Obrador Web".** Single highest-leverage move for ChatGPT/Gemini/Perplexity entity disambiguation. Zero personal info needed.
*Fix:* Create Wikidata Q-item (agency, Madrid, web design, founded 2026). 30 min.

**5. No Google Business Profile.** Madrid local intent is the entire revenue thesis. Service-area business (no storefront), free, postcard or video verifiable. Affects Gemini, AIO, Bing Copilot.

**6. Sitemap was never manually submitted to Bing Webmaster Tools.** GSC was submitted 04-25 but BWT never. Bing/Copilot uses BWT priority queue.
*Fix:* Verify + submit + add IndexNow key file. ~30 min.

**7. No legal pages (privacy / terms / cookie).** Spanish/EU law risk + Trustworthiness gap. Visible "Política de privacidad" link absent from footer.

**8. `sameAs` is one platform deep (only Instagram).** AI entity graph cannot triangulate.
*Fix:* Add GitHub org (xxyreN), Wikidata Q-id once created, Pascal mockup URL; eventually Clutch/Sortlist listings.

## Medium Priority Issues (Fix Within 1 Month)

**9. Demo pages /demos/cierzo-* are an AI hazard.** They look like client work in CreativeWork schema but are fictional. Risk: AI hallucinates them as references.
*Fix:* Add `"description": "Sitio demostrativo ficticio"` to each schema, or insert a visible "DEMO FICTICIO" line in JSON-LD descriptions, or `noindex` if you don't want them ranking at all.

**10. No `Service` entities, only Offers.** AI models index Services more cleanly than bare Offers nested in OfferCatalog.
*Fix:* Wrap each Offer in a `Service` entity with `serviceType`, `audience`, `areaServed: Madrid`, `provider: { @id: #organization }`.

**11. Process not marked up as HowTo.** "De idea a web en 3 pasos" is a textbook HowTo with 3 HowToSteps. AI assistants cite step-lists heavily.

**12. Blog citations bare.** 1,850-word pricing post makes claims with no linked sources, no methodology footer. Hurts Expertise.
*Fix:* Link INE Madrid hospitality data, Forum Café / Cafetología industry reports, Google CWV thresholds. Add "Cómo recopilamos estos datos" footer (visited N cafés, reviewed M quotes).

**13. No `knowsAbout` array on Organization.** Topics like "diseño web cafeterías", "carta digital", "SEO local Madrid", "café de especialidad", "Snipcart", "Stripe", "bilingüe ES/EN" should be enumerated for AI topical anchoring.

**14. English content invisible to AI crawlers.** Bilingual via JS toggle on single URL means crawlers only see Spanish. EN-speaking specialty coffee owners (real Madrid segment) are invisible.
*Fix:* Ship `/en/` mirror routes for at least home + compara + 1 blog post with hreflang pairs, OR accept the trade-off — the audit doesn't penalize since user already confirmed ES is the real market.

**15. No security response headers** (HSTS, CSP, XFO, XCTO, Referrer-Policy, Permissions-Policy). GitHub Pages caps this. Cloudflare in front would unlock all six.
